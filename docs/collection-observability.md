# Collection observability

OKEngine treats absence of collection telemetry as unknown, not as a successful
zero-yield run. Connectors register their configured sources and append one
attempt record for every source invocation, including failures, conditional
`not modified` responses, and successful runs that produce no new records.

## Runtime contract

The default deployed store is `/opt/data/collection/`:

- `sources.json` is the current configured-source registry, merged by stable
  `source_id` across connectors.
- `attempts-YYYY-MM.ndjson` is the append-only attempt ledger. Writes use an
  exclusive lock and one `O_APPEND` system call per JSON record.
- `wiki/operational/collection-health.md` is a derived projection generated by
  `collection_health.py` and displayed in Cockpit's Ops tab.

Each attempt carries connector/source identity, start and finish timestamps,
opaque input/output checkpoint hashes, fetched/extracted/accepted/rejected/
deduped/dead-letter counts, latency, a bounded error category, newest
publication time, and publication-to-ingest latency. A connector must emit an
attempt even when every count is zero.

Source declarations may classify `source_kind` as `primary` or `secondary` and
`independent_origin` as true or false. Missing declarations remain `unknown`.
For OPML feeds these are optional `sourceKind` and `independentOrigin`
attributes on the `<outline>` element.

## Retention and privacy

Attempt records are retained for 90 days by default
(`COLLECTION_LEDGER_RETENTION_DAYS`); the dashboard generator removes whole
monthly segments older than the retention boundary. The source registry is
retained while configured so never-run sources remain visible.

Telemetry must never contain credentials, authorization headers, request or
response bodies, licensed content, or private query text. Source identities are
stable hashes of native configuration values; checkpoints are SHA-256 digests.
Only bounded error categories are recorded—not exception messages that could
contain URLs or secrets. Restrict the runtime directory to the gateway UID;
attempt segments are created mode `0600`.

The ledger is operational evidence, not intelligence content. It is not copied
into source pages and should not be published with a vault snapshot.
