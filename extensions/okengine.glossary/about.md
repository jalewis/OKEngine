# Glossary — defined terms

**What this is.** Each page here is one **term**: a short, authoritative definition of a piece
of domain jargon, an acronym, or a concept the rest of the vault uses but never stops to explain.

**Why it's useful.** Vault pages are written for an expert reader and lean on shorthand. The
glossary is the layer that lets someone follow them without already knowing every term — and it's
built from the vault's *own* gaps: the lane finds terms that several pages reference but nothing
defines, and writes a grounded definition (or defers when it can't be grounded — it never invents).

**What you'll see on each page.**
- **term** — the human-readable name
- **definition** — 1–3 sentences, grounded in how the vault actually uses the term
- **aliases / acronyms** — other names it appears under
- **see_also** — the entities/concepts where the term shows up, so you can jump to it in context

These are definitions, not analysis — a reference layer under the knowledge, kept in sync as new
undefined terms accumulate.
